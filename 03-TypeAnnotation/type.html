Type Annotation Basics

We can tell TypeScript all sorts of things. 'This function returns a boolean'. 'This function accepts two numbers and returns a number.' 'This object must have a property called colors, and it must be set to an array of strings.' 'This variable is a string.'

You can run the file from any terminal, including the integrated terminal from vscode. tsc Name_of_file.ts will compile the file down into JS. It will do this by default EVEN IF THERE ARE ERRORS. This is tweakable.

Variables

After a variable name, assign a basic type by adding : type. CTD. declaration variableName: type = assignment;

Type Inference

Most of the time, the above syntax is overly-explicit and not needed.

Type Inference refers to the TypeScript compiler's ability to infer types from certain values in your code. TS can remember a value's type even if you don't provide a type annotation, and it will enforce that type in the future.

Any

any is the escape hatch! It turns off type checking for this variable so you can not get accosted by TS if you so desire. Don't overuse it, that entirely defeats the point of TS.

Delayed Initialized and Implicit Any

In the scenario where you need to declare a variable without assigning anything to it, TS will infer that that variable has a type of any.
