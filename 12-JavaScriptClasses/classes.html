JavaScript Classes

Classes are templates for creating object in JavaScript. They contain a few different important pieces which allow for creation and extension of customizerd and nicely organized objects.

Private Fields

Right now any property on any instance of the class is accessible to anyone with experience. Sometimes you see a leading underscore as a way of marking a property as private, but it doesn't actually stop them should they want to change it anyway.

But any property that starts with a # tells JavaScript that any property with that should only be accessible from the class itself, not the instances.

We can also make methods private!

Getters and Setters

These are bot pieces of syntatic sugar that sort of looks like a property and acts like method without actually existing on the class or the instance.

CTD. To assign it, use the get keyword before the method definition.

The set keyword will allow us to set the value of a private method. Wraps the property or method in some extra logic.

Static Properties and Methods

The static keyword makes it so that properties or methods on a class with that keyword exist only on the class itself, not on the instances themselves.

Inheritance

In JS, your classes can inherit stuff from another class. You can build new classes that inherit with the extends keyword.

Super Keyword

The super keyword comes into play when you have another class extending from another class, and you have a constructor in that child class.

If not specified, the child class gets the same constructor as the class it extends from. But the super keyword in the constructor will put specified parameters into the base class, and the rest will fall into the child class.
