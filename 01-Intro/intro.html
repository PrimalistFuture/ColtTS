TypeScript Intro

JavaScript has a lot of quirks. It has grown far beyond its original intent and a lot of issues have popped up because it is bursting at the scenes, as it were. Lots of fixes and enhancements, but some of its core issues have been amplified.

JavaScript really doesn't like throwing errors as part of its design ethos. And that is a defensible position, but it can lead to a lot of bugs.

TypeScript aims to fix some of these issues. It is built on top of JavaScript - a superset. The main feature of TS is that it adds types to JavaScript syntax.

TypeScript has built in static checking, which attempts to detect errors in our code without running it. JavaScript will still execute the code, TS will not. TS does this error checking on the basis of the 'kinds of data' in our program AKA types.

This can matter a lot because it helps us find errors, even before the code actually runs. Analyzes our code as we type. After TS clears us, it will pass the code off to JS to compile and execute. So any JS syntax is perfectly valid in TS. Though some of it will not exist in the compiled JS code.
