The Lib Compiler Option

We saw that TS knows about the document object and has a huge list of its seperate definitions. But we can also turn that off so that it doesn't get in our way. And at the same time, we can also tell TS about other types and whatnot it may not know.

If we turn the lib option on in the tsconfig.json, we are telling TS what it needs to know, and getting rid of the default behavior of it knowing the DOM and whatnot.

Files that end in .d.ts are where various types are defined.

TS Non-Null Assertion Operator

When we use document.getElementById or similar document method and save it to a variable, it will usually return a predefined HTMLElement type or null if it couldn't find it. TS will get mad at us if we try and do stuff that variable, because we might be trying to do something to null, which never works.

The JS solution is to add a ? after the variable (Check index.ts), or we can add a ! to after the entire method (CT index.ts).

Type Assertion Syntax

Sometimes, you might have more specific information about a value's type and you want to make sure TS knows it too. You can assert a value's type by using the as keyword, followed by the specific type you want to assert.

Check the index.ts for examples and syntax.

Be careful though, the as keyword doesn't actually change the type of the thing, we are just telling TS to treat it as another type for now.
