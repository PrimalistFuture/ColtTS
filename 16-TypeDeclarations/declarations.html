Type Declarations

This section is about working with 3rd Party JavaScript libraries in a TypeScript project. We ran npm init -y in this folder, and then npm install axios.

Declaration review

We have seen in the mini project that files that end in .ts are implementation files that contain files that produce .js outputs and are where you would normally write your code. .d.ts files are declaration files that contain only type information - no logic whatsoever.

Remember that if we want to add another .d.ts file, we need to add it to the lib part of the tsconfig.json we get from tsc --init.

Some libraries come with their own .d.ts type declaration files. Axios did, we could find and look through it in the node_modules folder in the 'axios' file.

Lodash did not have its own .d.ts file that came with it. That means we would have to build one, but for just about every library that anyone uses in JavaScript, someone has probably already made a .d.ts file for it. We just have to find it and install it.

A library on GitHub, DefinitelyTyped, has .d.ts file support for over 7k of the most popular JS libraries.

To get one such .d.ts file for lodash, we ran npm install --save-dev @types/lodash

The TypeScript docs also have a search of most of the libraries that have .d.ts files for them.
