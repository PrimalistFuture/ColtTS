TypeScript Compiler

This section is a small break from TS syntax to talk about the TS compiler. We will be configuring out settings and whatnot.

Create the TS config file with tsc --init in the folder you want to be make a tsconfig.json file. THIS IS A JSON FILE SO YOU NEED TO USE DOUBLE QUOTES, NOT SINGLE QUOTES.

Remember to compile your TS into JS you need to use tsc NAMEOFFILE.ts, which will then make a .js file with all of the TS bits removed.

Watch Mode

The tsc -w NAMEOFFILE.ts will 'watch' a file for changes and automatically recompile when something changes. This will give you error information in your terminal.

Multiple Files

If we have multiple files to compile, of course we could do them individually. But if both files are found in the same folder, the command tsc by default will just compile any .ts files it comes across.

The Files Compiler Option

If you want to tell tsc to compile a certain file by default, you can change that in the tsconfig.json file, by adding it to a files array outside of the compileOptions folder.

We can also tell TS to include folders, which will recursively look through all of the files and add them. The exclude keyword then tells TS to ignore certain files or folders that would otherwise be included. By default, exclude will not look in your node_modules folder. So if something is excluded, make sure to add back in "node_modules" so TS doesn't go burrowing down into that.
